## 시스템 개요

모바일 출입카드(패스) 발급 및 관리 웹시스템. 두 가지 역할 기반 인터페이스 제공.

---

## 고객사 관리자 (Manager, /manager)

### 카드 발급 관리 (/manager/card-requests)
- 백엔드 외부 API를 통해 카드발급 요청 발생
- 발급 요청 목록 표시, 상태: **대기, 승인**
- 대기 중인 요청은 승인 또는 거부 가능
- **거부 시 DB에서 데이터 삭제** (confirm 확인만, 사유 입력 없음)
- 승인된 요청은 상세 정보 확인 가능
- 카드 상세: 사용자 정보, 요청/승인 정보, 카드 정보(번호, 유효기간, 상태)
- **활성화번호**: 카드가 모바일에 미등록 시에만 표시, 등록 완료 시 숨김
- 승인된 카드는 모바일 활성화/미활성화 상태를 뱃지로 구분

### 관리자 (/manager/users)
- 관리자 목록 및 초대 (이메일 기반 JWT 링크)
- **주관리자** (1명, 고객사 대표) / **부관리자** 구분
- 주관리자만 초대 가능
- 주관리자는 다른 관리자에게 주관리자 권한 위임 가능

### 크레딧 관리 (/manager/credits)
- 1 크레딧 = 1 카드 생성
- ~~충전 요청/충전 코드 입력~~ → **삭제됨**
- **슈퍼관리자가 직접 충전하는 방식으로 변경**
- 페이지 구성: 잔여 크레딧 현황 카드 + 크레딧 사용/충전 이력 테이블

### 카드 디자인 (/manager/card-design)
- 모바일 카드 이미지 선택 (프리셋)
- 고객사별 커스텀 카드 이미지 2개 추가 등록 가능
- *(미구현)*

### 설정 (/manager/settings)
- 사업장 이름 (변경 가능)
- **라이센스 코드** (변경 불가, 기존 "발급 코드"에서 명칭 변경)
- 관리자 이메일 (주관리자 이메일)
- 생성일

## 슈퍼관리자 (Admin, /admin)

### 고객사 관리 (/admin/clients)
- 고객사 목록 및 생성 (초대 방식)
- 생성: 고객사명 + 초기 크레딧 + 관리자 이메일 → 라이센스 코드 메일 발송
- 로그인 화면에서 회원가입 → 라이센스 코드 입력 → 고객사 인증 → 추가 정보 입력 → 고객사+최초 관리자 생성

### 크레딧 관리 (/admin/credits)
- ~~충전 요청 탭/코드 발급 시스템~~ → **삭제됨**
- **사업장 드롭다운 선택 + 수량 입력 → 즉시 충전 방식으로 변경**
- 충전 이력 테이블 유지

### 관리자 (/admin/users)
- 백오피스 관리자 목록 및 초대 (이메일 기반)
- 고객사 소속 아닌 독립 관리자 (URL에 이메일만 포함)
